# ProGrok 协议注册工具

ProGrok 是一个面向 Windows 的本地 Web 注册工具，可批量执行账号注册、保存认证文件、手动测活，并按需导入 CPA 或 Sub2API。服务默认只监听本机地址，不依赖 PostgreSQL、Redis或其他管理后台。

本项目基于 [HM2899/grokcli-2api.git](https://github.com/HM2899/grokcli-2api.git) 的 **注册功能进行开发的状态机**。

> 请仅在你有权使用的账号、邮箱、代理和服务上运行本工具，并遵守相关网站与服务商条款。注册结果、邮箱凭据和管理密钥均由使用者自行保管。

## 系统要求

- Windows 10 或 Windows 11，推荐 64 位系统。
- 可正常访问 Python 软件源、Camoufox 下载源及注册目标站点的网络。
- 推荐 Python 3.12；支持 Python 3.10 及以上版本。
- 首次安装需要约数百 MB 磁盘空间，用于两个 Python 虚拟环境和 Camoufox 浏览器。
- 默认端口：Web 管理页 `3080`，本地 Turnstile Solver `5072`。


## 一键安装并启动

解压发布包后，双击：

```text
install_and_start.cmd
```

脚本会自动完成以下工作：

1. 检测 Python 3.10 或更高版本。
2. 若没有 Python，优先通过 Windows Package Manager 安装 Python 3.12；不可用时改用 Python 官方安装程序。
3. 创建项目和本地过盾服务所需的虚拟环境。
4. 安装全部 Python 依赖。
5. 下载 Camoufox 浏览器运行环境。
6. 启动本地 Turnstile Solver 和 ProGrok Web 服务。

启动完成后访问：

<http://127.0.0.1:3080>

首次安装耗时取决于网络速度。命令窗口出现下载或安装进度时不要关闭。

安装成功或失败后窗口都会保留，并等待按键后关闭。如果启动失败，请根据窗口末尾的红色错误信息排查或截图反馈。

如果系统策略阻止 PowerShell 脚本，可在 PowerShell 中执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

然后重新双击一键启动脚本。

## 日常启动和停止

完成首次安装后，日常使用可直接双击：

```text
start.cmd
```

停止由本项目启动的后台进程：

```text
stop.cmd
```

也可以在 PowerShell 中运行：

```powershell
.\start.ps1
.\stop.ps1
```

重复启动时脚本会检测现有服务，不会故意重复占用相同端口。

## 首次配置

打开页面后，先完成配置，再点击“保存配置”。敏感字段会保存在本机的 `config.json` 中。

### 注册数量、并发和错峰

- 注册数量：单批支持 `1-10000` 个账号。
- 并发数：支持 `1-10`，表示同时运行的注册任务数量。
- 错峰毫秒：每个注册任务之间的启动间隔，范围 `0-60000` 毫秒。

并发越高不一定越稳定。邮箱、代理或目标链路出现限流时，应降低并发并增加错峰时间。

### 邮箱服务

邮箱类型只保留两种：

- YYDS Mail：可前往 [https://vip.215.im/](https://vip.215.im/) 注册账号，并在平台生成邮箱域名和 API 密钥；填写密钥后，域名也可留空，由服务动态选择可用域名。
- 自定义（Cloudflare 邮箱）：所有字段默认留空，由用户填写自己的兼容邮箱服务。

自定义邮箱需要填写：

- API 地址：邮箱服务的根地址，例如 `https://mail.example.com`，不要填写具体接口路径。
- API Key / 管理员密钥：用于创建邮箱和读取邮件的密钥。
- 邮箱域名：填写该服务允许使用的域名；服务支持自动返回域名时可尝试留空。

当前自定义适配针对 Cloudflare Temp Email 兼容接口，包括创建地址、获取域名和读取邮件。其他邮箱渠道只有在接口兼容时才能直接使用，否则需要额外适配。

发布包不包含开发者本机的自建邮箱地址、域名、API 地址、管理员密钥或凭据文件，也不会为“自定义”自动填入任何本地资料。

邮箱前缀由程序为每个账号自动生成随机值，批量任务不会使用固定前缀。

### 过盾方式

本地 Turnstile Solver 为默认选项。一键启动会同时启动 Solver，页面会自动检测本机 `5070-5090` 端口，并优先使用默认的 `5072`。

如果页面显示 Solver 未在线：

1. 等待首次 Camoufox 下载完成。
2. 点击“重新检测”。
3. 确认端口 `5072` 没有被其他程序占用。
4. 查看 `logs` 目录中的错误日志。

也可以选择 YesCaptcha，并填写自己的 YesCaptcha Key。使用外部api服务可能产生费用。

### 代理池

页面打开时，如果代理池为空，程序会自动尝试读取 Windows 系统代理和代理环境变量；仍未找到时，会探测常见本地代理监听端口。也可以点击“检测本机代理”重新检测。检测结果只会填写到页面，点击“保存配置”后才会持久化，已有代理内容不会被自动覆盖。

本地 Turnstile Solver 会以代理支持模式启动。保存配置或开始注册时，当前代理池会自动同步给 Solver，使注册请求和验证码浏览器共同使用代理；代理池为空时两者均保持直连。

代理池支持每行一个代理，允许以下形式：

```text
http://127.0.0.1:7890
http://user:password@host:port
socks5://host:port
host:port
```

如果多条代理共用同一用户名和密码，可以在单独的代理用户名、代理密码字段中填写。

轮换策略：

- 轮询：按顺序循环使用代理。
- 随机：每次随机选择代理。
- 固定：同一任务尽量保持使用同一代理。

留空表示直连。代理不可用会直接影响注册、邮箱访问或后续测活。

## 注册、测活和导入流程

默认流程为：

```text
注册成功 → 保存本地账号 → 按配置自动导入
```

测活改为手动触发，不会阻塞注册或自动导入。注册失败、取消、协议异常或没有生成本地账号文件的记录不会进入导入队列。

监控区分别显示注册、测活和导入状态，包括成功、失败、进行中、排队数量与注册用时。

### 暂停、继续和重置

- 暂停：停止继续创建新注册任务，等待当前正在执行的任务结束。
- 继续：按原批次配置处理剩余数量。
- 重置本轮任务：清除页面上的本轮监控记录，不删除已经生成的账号文件；重置后不能继续被清除的暂停批次。

### 手动测活

注册完成后，可在测活卡片点击“开始测活”。失败后可点击“重试测活”。

- 测活并发数：每一批同时处理的账号上限，范围 `1-10`。
- 测活错峰毫秒：同批账号依次发起请求的间隔，默认 `10000` 毫秒。
- 测活模型：填写用于验证账号状态的模型名称。

### 自动导入

自动导入默认关闭。开启后选择 CPA 或 Sub2API，并填写自己的服务器信息。

选择 Sub2API 时：

- 填写服务根地址，不要只填写某个管理页面路径。
- 认证方式可选“管理员邮箱 + 密码”或“管理员 API Key”。
- 使用邮箱密码方式时，必须填写 Sub2API 管理员邮箱和密码。

选择 CPA 时：

- 填写 CPA 服务根地址。
- 填写 CPA 管理密钥。

“默认注册 JSON”与自动导入对象会互相关联：选择 CPA 时生成 CPA JSON，选择 Sub2API 时生成 Sub2API JSON。不要把两种格式混用。

- 导入并发数：每一批同时处理的账号上限，范围 `1-10`。
- 导入错峰毫秒：同批账号依次导入的间隔，默认 `10000` 毫秒。

导入失败后可在监控卡片点击“重试导入”。首次失败不会阻塞后续完整批次。

### 手动导入 JSON

点击配置卡片右上角的“手动导入 JSON”，可一次选择一个或多个 JSON 文件。导入前页面会提醒先检查下方的 CPA 或 Sub2API 连接信息。

文件内容必须与当前选择的导入对象匹配。手动导入只处理所选文件，不会把注册失败记录加入队列。

## 账号文件和下载

注册成功后，独立账号文件保存到 `data\accounts`，合并认证数据保存到 `data\auth.json`，注册产生的 SSO 记录保存到 `data\register_sso`。

页面支持按批次或全部成功账号下载：

- 纯 SSO
- `sso=...` Cookie 格式
- 邮箱 + SSO
- 邮箱:密码:SSO
- 普通 JSON（邮箱 + 密码）
- CPA JSON
- Sub2API JSON

下载只包含成功生成账号的记录，不包含注册失败账号。

## 配置保存说明

- 点击“保存配置”后，页面配置会写入本机 `config.json`。
- 注册数量、注册并发、测活并发和导入并发会随配置持久化。
- 邮箱 API Key、代理密码、CPA 管理密钥和 Sub2API 凭据同样会保存在该文件中。

## 常见问题

### 双击一键启动后立即关闭

右键在终端中运行 `install_and_start.cmd`，查看具体错误。常见原因是安装程序被安全软件阻止、网络无法访问 Python 官网或磁盘空间不足。

### Python 已安装但仍提示找不到

关闭并重新打开终端后再运行。也可以设置当前终端使用的 Python：

```powershell
$env:PROGROK_PYTHON = "C:\Path\To\Python\python.exe"
.\start.ps1
```

### 端口被占用

检查 `3080` 和 `5072` 是否被其他程序使用：

```powershell
Get-NetTCPConnection -LocalPort 3080,5072 -ErrorAction SilentlyContinue
```

停止冲突程序，或先运行 `stop.cmd` 停止由本项目记录的后台进程。

### Solver 一直未就绪

首次启动需要下载 Camoufox。确认网络畅通后再次运行 `start.cmd`，脚本会继续补齐缺失环境。

### 注册频繁失败或链路阻塞

- 降低注册并发。
- 增加注册错峰毫秒。
- 检查邮箱 API 是否限流或能否正常收到验证码。
- 检查代理稳定性；必要时先使用单个可靠代理验证。
- 测活和导入保持默认 10 秒错峰，确认稳定后再逐步缩短。

### 页面刷新后配置丢失

修改后必须点击“保存配置”。同时确认项目目录有写入权限，且安全软件没有拦截 `config.json`。

### 日志位置

Web 服务标准输出和错误信息保存在 `logs` 目录。排错时可查看：

```powershell
Get-Content .\logs\app.err.log -Tail 100
Get-Content .\logs\app.out.log -Tail 100
```

分享日志前请先检查并删除邮箱、代理、Token、Cookie、SSO 和 API Key 等敏感内容。
